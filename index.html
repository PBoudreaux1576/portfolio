04 — PERSONAL PROJECTS

Dual-Database Inventory Lookup

A single interface that answers, "what do we actually have?" - built on my own initiative.

The company's inventory was split across two systems, Atrex (ElevateDB) and BreezeERP (SQL Server), so there was no single place to check stock, pricing, sales history, or purchase orders.

◆
Two front ends

Runs as a Flask web app in the browser and inside Claude Desktop through an MCP server. The web app shows an inventory card , on-hand, available, on-order, allocated, pricing. It has sub-tabs for sales history, purchase orders, and receiving, plus a date filter and branch switcher.

◆
Engineering detail

Shared query logic is reused across the web and MCP servers, with field names normalized so one table renderer handles both systems. The browser version fetches inventory and history in parallel so a slow query doesn't block the card.

◆
AI integration

The MCP server lets an AI assistant query live inventory directly.  Applying the Claude certification work to a real operational tool.

MTG - The Musical

Real-time reactive music engine · Personal project
2026

An application that listens to a game of Magic: The Gathering in progress and generates reactive music reflecting the colors, themes, and events of the game as it unfolds. Each of MTG's five colors carries a distinct emotional identity that maps to sound design, so game events drive the music in real time.  A Green creature entering should feel ancient and organic; a Red board wipe should feel explosive.

Architecture

The input source and the music engine are separate components connected by an abstract event interface. The input (a digital game client today, a camera-based vision pipeline in a future version) can change without touching the engine.

Stack

C# / .NET 10, with FMOD for layered adaptive audio (pad, rhythm, lead, stingers). Game state is observed over a protobuf protocol; card metadata is resolved against a cached Scryfall dataset.

Status

The adapter, event boundary, and music engine are all built and live-validated. Real card events flowing into audible, reactive audio against a live game. Tuning and a future camera-input mode are the next phases.

05 — STARTUP

ClasScape

An education-technology venture I founded and led.  From a concept to a funded, tested prototype.

Company Goal

ClasScape was a gamification tool for student assessment, built for VR, PC, and Mac. Teachers would create quizzes through a web interface; those questions appeared inside a game, and students earned a short gameplay session afterward, with rewards tied to their performance. The goal was to raise student engagement and retention without adding work for teachers.

My role

Founder and Entrepreneurial Lead. I led a small team and worked with academic mentors (Dr. Ryan Walker of MSU) and an industry mentor (Vince Jordan of Lobaki). I built the VR assessment demo using Unreal Engine 4 and WordPress, wrote and presented the grant materials, and ran school demonstrations and market research.

Work completed

A functional VR demo of the assessment portion, tested in two schools and shown at the Innovate MS Accelerate conference. The demo pulled questions from a Firebase NoSQL database and presented them in random order. And a working WordPress Site for teachers to put in the questions. The project reached an alpha-prototype stage.

Why it Ended

The COVID-19 pandemic in early 2020 disrupted in-person school demonstrations, which were central to investor engagement and grant-supported pilot deployments, and significantly impacted the startup’s ability to sustain momentum, leading to it winding down by January 2022.

What it demonstrates

I took an idea from concept to a funded, tested prototype: writing the grants, building the software, securing mentors, and running real customer discovery.
